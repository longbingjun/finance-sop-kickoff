# 财务SOP启动访谈（finance-sop-kickoff）

## 何时使用

用户显式点名调用本 skill，或明确要求"梳理这个项目的SOP / 沉淀业务知识 / 做知识交接文档"时加载。不要根据项目文件夹里有Excel就自动触发。

## 硬性约束（先读，优先级最高）

**禁止**

- 把推断/猜测当作已确认口径写入 03，未确认必须标"待确认"并注明谁能确认
- 自己脑补多字段组合的 AND/OR 逻辑——必须由用户明确说出
- 把"应该是……吧"这类不确定表述当成确认记录
- 产出清洗脚本或直接处理业务数据（不是本 skill 职责，只做知识蒸馏）
- 在 SKILL.md 或产出文档里夹带其他项目的业务细节——本 skill 是通用方法论

**必须**

- 每条字段/口径/软知识记录标注确认人和日期
- 每次调用先检查断点续跑状态，不重复问已确认内容
- 第2轮环节拆解先由 agent 给候选草案，不要求用户从零构建
- 高优先级字段的回答要检查是否覆盖全部维度，缺项单独追问

## 断点续跑

调用时先检查当前项目文件夹是否已有 `00-体系地图.md` ~ `05-工具操作手册.md`。若有，汇报进度、问从哪继续；若无，从 [templates/](templates/) 复制模板，从第1轮开始。

## 总流程

| 轮次 | 产出 | 方式 |
|------|------|------|
| 第1轮 | `00-体系地图` + `01-背景与目标` | 开放式访谈 |
| 第2轮 | `02-环节与交付标准` | agent 生成候选拆分草案 → 逐条确认 |
| 第3轮（每环节一次） | `03-业务口径说明` + `04-软知识与可信度台账` + `05-工具操作手册` | 扫描该环节输入数据 → 分级提问 → 机会式捕获 |

---

## 第1轮：体系地图与背景目标

按顺序问，不要一次抛全部问题：

1. 这项工作解决什么问题？做错了后果是什么？
2. 下游是谁（谁用结果）？上游是谁（输入从哪来）？
3. 这项工作在更大流程里处于哪个位置？
4. 有没有历史文档、旧口径、遗留待确认事项？

写入 [templates/00-体系地图.md](templates/00-体系地图.md)、[templates/01-背景与目标.md](templates/01-背景与目标.md)。

---

## 第2轮：环节拆解

**不要直接问"你的工作分几步"**——多数业务人员答不完整。改为 agent 生成候选草案，用户只需确认。

1. 请用户按实际顺序自由叙述全流程；同时扫描项目文件夹（文件清单、Excel sheet 命名、粗略列结构）。
2. 提取候选切分信号（6类信号见 [reference.md](reference.md) §1），生成候选草案表：暂定名称 / 判断依据 / 大致输入 / 大致输出。
3. 逐条确认，每条只问三个封闭问题：
   - 名字准不准确？
   - 要不要拆细/合并？（给出具体理由）
   - 结束标志（交付物/检查点）是什么？
4. 收敛规则：**该拆**——独立交付物 / 跨责任人或系统 / 统计粒度变化；**不该拆**——同一张表内连续操作、无中间交接。
5. 写入 [templates/02-环节与交付标准.md](templates/02-环节与交付标准.md)：每行环节记录输入、输出、交付标准、存放位置、同步对象、时效。

---

## 第3轮：逐环节字段/口径/软知识/操作蒸馏

**一次只做一个环节**，做完确认再进入下一个。

1. 扫描该环节输入数据：

   ```bash
   python scripts/scan_fields.py "<该环节输入文件路径>" --output <结果.md>
   ```

   （Windows 上避免用 `>` 重定向看结果，用 `--output` 直接写文件，见脚本内说明）

   分级规则见 [reference.md](reference.md) §2。扫描不了则退化为人工询问字段清单。

2. **高优先级字段**：合并式提问，一次问完但检查覆盖度，缺项追问：

   > "字段【X】：实际数据是 [扫描结果]。含义是什么、从哪来、谁负责、正常范围/合法取值是多少、越界算什么情况？"

3. **中优先级**：简化问含义和取值。**低优先级**：批量确认，不逐条追问。

4. **口径组合逻辑**（字段问完后）：

   > "这个环节里，什么样的一行算要处理的？各条件是AND还是OR？有没有例外？"

5. **机会式捕获**（贯穿2-4，不单独开轮次）：
   - 字段/规则不显然 → 追问为什么 → 记入03备注
   - 用户提到"以前也出现过" → 追问怎么处理、找谁核实 → 记入04
   - 用户带出具体操作动作 → 直接记入05，不追加提问

6. 汇总该环节新增内容，用户确认后进入下一环节。
